    6.Descendant Combinator

        The descendant combinator matches all elements that are descendants of a specified element.

        The following example selects all <p> elements inside <div> elements: 

        div p {
        background-color: yellow;
             }

             descendant means all element of specified element inside the any element 
             
             for eg: Descendant.html (2.Selectors)




    7.Child Combinator (>)

        The child combinator selects all elements that are the children of a specified element.

        The following example selects all <p> elements that are children of a <div> element:         

        div > p {
        background-color: yellow;
                }

        It will select only child element not descendant means 

            <div>
                <p>This is the child element </p>

                <section>
                    <!-- not Child but Descendant -->
                <p>This is not child element of div , it is the child element of section so it will be not selected </p>
                </section>

            </div>

        9. Text-shadow

            text-shadow: offset-x offset-y blur-radius color;
            text-shadow: 2px 2px; /*Only offset-x and offset-y */
            text-shadow:2px 2px red; /*Only offset-x and offset-y and color red */
            text-shadow:0px 0px 5px; /*with only blur 5px */
            text-shadow: 1px 1px 2px black, 0 0 25px blue, 0 0 5px darkblue; // Multi shadows
            text-shadow: 1px 1px 2px 100px red; // 100px used for spread of shadow in all direction.
            text-shadow: inset 10px 12px; // for shadow inside the box.

            *offset-x : Horizontal distance of the shadow.
                    - Positive value moves the shadow to the right.
                    - Negative value moves the shadow to the left.

            *offset-y : Vertical distance of the shadow.
                    - Positive value moves the shadow downward.
                    - Negative value moves the shadow upward.

            *Blur-radius(optional) : The amount of blurring applied to the shadow.
                    - 0 creates a sharp shadow.
                    - Higher values make the shadow softer.
                    - Default value: 0 (no blur).

            *color(optional) : The color of the shadow.
                    -Can be any valid CSS color format (red, #hex, rgb, rgba, etc.).
                    -If omitted(left), the shadow color defaults to the text color.     

            *inset : This keyword is used for creating shadow inside the box or text.                           


            *****Note: We can give only blur also by giving offset-x and offset-y 0px and only blur property.

            *** Animation ***

An animation lets an element gradually change from one style to another.You can change as many CSS properties you want, as many times as you want.

To use CSS animation, you must first specify some keyframes for the animation. Keyframes hold what styles the element will have at certain times.


    -->>The @keyframes Rule

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style at certain times.

        syntax->                 @keyframes animation-name {
                                                     from 
                                                     {
                                                        background-color: red;
                                                        }
                                                     to 
                                                     {
                                                        background-color: yellow;
                                                        }
                                                    }
                example: Keyframe.html (12.Animation)
    
    -->To get an animation to work, you must bind the animation to "an element"(SINGLE ELEMENT).

            
            
            Animation Have various properties are as follows:

1.@keyframes --> hold what styles the element will have at certain times.

2.animation-name --> It is an identifiers for the keyframe to choose that specific element to apply animation on it.

3.animation-duration --> The animation-duration property defines how long an animation should take to complete. If the animation-duration property is 
                         not specified, no animation will occur, because the "default value is 0s" (0 seconds). 

4.animation-delay --> The animation-delay property specifies a delay for the start of an animation.

5.animation-iteration-count --> The animation-iteration-count property specifies the number of times an animation should run.

6.animation-direction --> The animation-direction property specifies whether an animation should be played forwards, backwards or in alternate cycles.
                    
        --->>> ****The animation-direction property can have the following values: **** <<<---

* normal - The animation is played as normal (forwards). This is default  (1 → 100%).

* reverse - The animation is played in reverse direction (backwards) ( to se from(keyframe wala ) jayega means ulta)  (100% → 1%).

* alternate - The animation is played backwards first, then forwards (Means start hoke destination pe jake phir start point pe) Note: return hoke tabhi
              ayega jab iteration-count 1 se bada hon kyuki uss time dusri baar n animation hon raha hain bas keyframes change hongayein hain (1 → 100%, phir 100% → 1%).

* alternate-reverse - The animation is played forwards first, then backwards (same alternate wala but to(Keyframe wala)se start honga) and 
                      condition same iteration wala jo upar hain iske. (100% → 1%, phir 1% → 100%)

7.animation-timing-function --> The animation-timing-function property specifies the speed curve of the animation.

        --->>> ****The animation-timing-function property can have the following values: **** <<<---

    * ease - Specifies an animation with a slow start, then fast, then end slowly (this is default)
    * linear - Specifies an animation with the same speed from start to end.
    * ease-in - Specifies an animation with a slow start.
    * ease-out - Specifies an animation with a slow end.
    * ease-in-out - Specifies an animation with a slow start and end.
    * cubic-bezier(n,n,n,n) - Lets you define your own values in a cubic-bezier function.

8.animation-fill-mode --> The animation-fill-mode property specifies a style for the target element when the animation is not playing 
                            (before it starts, after it ends, or both).

    **Note : CSS animations do not affect an element before the first keyframe is played or after the last keyframe is played. The animation-fill-mode 
             property can override this behavior.

        --->>> ****The animation-fill-mode property can have the following values: **** <<<---    

    * none - Default value. Animation will not apply any styles to the element before or after it is executing.
            Hindi-> Default: Animation se pehle aur baad mein koi style apply nahi hota.

    * forwards - The element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count) 
                Hindi-> Animation khatam hone ke baad, element last keyframe ki styling retain karega (jaise blue color aur 100px move hua tha).

    * backwards - The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during
                  the animation-delay period.
                  Hindi-> Animation start hone se pehle hi first keyframe ki styling apply ho jati hai (delay mein bhi dikhega).
                
                **Note: animation-fill-mode: backwards; ka kaam sirf animation start hone se pehle first keyframe (from ya 0%) ka style apply karna hota hai. 
                  Lekin animation complete hone ke baad ye style retain nahi karta. **eg-> animation.html & css (circle2 class in css file )**

    * both - The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions.
                Hindi-> Dono kaam karega: Pehle se styling apply + Khatam hone ke baad last styling retain.

        *** Transition ***

Transitions are used for simple state changes, like hovering over a button. They need a trigger, such as :hover, and they animate from one state to another. 
They’re straightforward with limited control—just start and end states. Animations, on the other hand, are more complex. They use @keyframes to define multiple steps 
and can loop or run automatically. They offer more control over the intermediate steps and timing.


**2 States ke Beech: Start state (normal) → End state (hover/focus). pe transition kaam karta hain, It always need trigger: Hover, click, JS event.

A transition in CSS is used to smoothly change a property from one state to another over a period of time. 

                                                    ** Without Transition (Instant Change) ** 
If you change the color of a button when you hover over it, the change happens instantly. The color will immediately switch to blue without any smooth effect.

                                                    ** With Transition (Smooth Change) **
If you add a transition, the color will change gradually over time Now, when you hover over the button, it will slowly change from red to blue in 1 second instead of 
changing instantly.                                                    


        Transition Properties are as follows:

**transition-property -> Which CSS property to animate? (e.g., width, background-color, opacity)

**transition-delay -> How long should the transition take? (1s, 0.5s, 2s)

**transition-timing-function ->  How the speed changes? (ease, linear, ease-in, ease-out, ease-in-out,cubic-bezier(n,n,n,n))

**transition-delay -> (optional) → When should the transition start? (0s, 1s, etc.)

                            **Shorthand**

          /* Apply to 1 property */  

            transition: transition-property  duration  delay(optional)  easing function ; 
            transition: margin-right 4s  1s ease-in-out ; 

         /* Apply to multiple property */ 

            transition: width 2s 5s linear, height 4s; 
            or transition: width 2s linear 5s, height 4s;
    
         /* Apply to all changed properties */  

            transition: all 2s ease;

    for eg: transition.html & transition.css (13.Transition)





        *** Transform ***

CSS transforms allow you to move (translate), rotate, scale (small or large the element size), and skew(tilt) elements.
With the CSS transform property you can use the following 2D transformation functions:

**translate() - The translate() function moves an element from its current position (according to the parameters given for the X-axis and the Y-axis).
                    transform: translate(50px, 100px); // Input values can be negative as well also.
                    Note: we can any css unit also ex. transform: translate(-100%, -100%); ex- em/rem/%/px etc.

**rotate() - The rotate() function rotates an element clockwise or counter-clockwise according to a given degree.
                    transform: rotate(20deg); //(clockwise rotation) , Input values can be negative(anti-clockwise)  also.

**scaleX() - The scaleX() function increases or decreases the *width* of an element.
                    transform: scaleX(2); // less than 1 till 0 to decrease the width and larger than 1 to increase the width.
                    (-)negative to invert the elemement and lesser the value you go in negative, larger the width becomes in inverted . 

**scaleY() - The scaleY() function increases or decreases the height of an element.
                    transform: scaleY(3); // less than 1 till 0 to decrease the height and larger than 1 to increase the height.
                    (-)negative to invert the elemement and lesser the value you go in negative,larger the height becomes in inverted .

**scale() - The scale() function increases or decreases the size of an element (according to the parameters given for the width and height).
                    transform: scale(x,y); all conditons applied of above scaleX() and scaleY().

**skewX() - The skewX() function skews an element along the X-axis by the given angle.
                    transform: skewX(20deg); // positive for right direction and negative for left direction tilt.

**skewY() - The skewY() function skews an element along the Y-axis by the given angle.
                    transform: skewY(20deg); // positive for down direction and negative for top direction tilt.

**skew() - The skew() function skews an element along the X and Y-axis by the given angles.
                    transform: skew(x,y) // all conditons applied of above skewX() and skewY().

**matrix() - The matrix() function take six parameters, containing mathematic functions, which allows you to rotate, scale, move (translate), and 
             skew elements. The parameters are as follow: matrix(scaleX(), skewY(), skewX(), scaleY(), translateX(), translateY())
                    transform: matrix(scaleX(), skewY(), skewX(), scaleY(), translateX(), translateY());
                    transform: matrix(1, -0.3, 0, 1, 0, 0);




Note:                 /* Multiple function values */

    transform: translateX(10px) rotate(10deg) translateY(5px);// order matters the first value will be primary all remaining property 
                will act on first property .






                    

                     ***Grid ***

The Grid Layout Module offers a grid-based layout system, with rows and columns.
The Grid Layout Module allows developers to easily create complex web layouts.
The Grid Layout Module makes it easier to design a responsive layout structure, without using float or positioning.
The CSS grid properties are supported in all modern browsers.

*Note: To apply Grid set Display:Grid or Display:inline-grid;  


     
                Grid Vs Flexbox

*The CSS Grid Layout should be used for two-dimensional layout, with rows AND columns.
*The CSS Flexbox Layout should be used for one-dimensional layout, with rows OR columns.


             CSS Grid Components

    A grid always consists of:
    
    *Grid Container: the parent (container) <div>element.
    *Grid Items: the items inside the container <div>


          **fr - Fractional Unit **
 
Fractional Unit is relative Unit which automatically divides the available space into fractions.
if I have to place 3 box in row-order and size of widht is 3000 px and I have size to all three box 1fr,1fr,1fr then each box will take 1000px;

and if 1st box- 1fr, 
       2nd box- 2fr,
       3rd box- 1fr ,       -> then 3000/4fr=> 750px per fr means 1fr will take 750px and 2nd box will take 1500px to render.

The fr unit can be used when defining grids, like any other CSS length such as %, px or em.
The fr unit stands for "fraction". This unit automatically divides the available space into fractions.
Example: 1fr will take 1 fraction of the available space, while 2fr will take 2 fractions of the available space.

              ****Grid Container ****

A grid container contains one or more grid items arranged in columns and rows.
Direct child elements(s) of the grid container automatically becomes grid items.
An element becomes a grid container when its display property is set to grid or inline-grid.

        display:grid;
        display:inline-grid;

There are various properties on Grid Container .

*1. Grid-template-rows
*2. Grid-template-columns
*3. grid-template-areas
*4. gap
*5. justify-items.// To change the behaviour "inside  cells for all cells", adjust the position of individual items "inside their grid cells".
*6. align-items. // To change the placement of content "inside" cell for all cells  to control their placement (e.g., center, start, end, stretch).
*7. justify-content // It is only used when total items spaces are smaller than container size. to adjust all cells respect to container
                                                                                                (all grid are in center of container)
*8. align-content // It is only used when total items spaces are smaller than container size.
*9. place-items // shorthand of align-items and justify-items ex-> place-items: align-items justify items 
*10. place-content // shorthand of align-content and justify-items ex-> place-content: align-content justify-content

        Note: To check Justify-content, Align-content, Justify-items, Align-items check 15.Grid(2.Justify_Content.html) second grid and its comment.

          *Grid-template-columns*

The grid-template-columns property defines the number of columns in grid layout, and it can define the width of each column.
The value is a space-separated-list, where each value defines the width of the respective column.
*If you want your grid layout to contain 4 columns, specify the width of the 4 columns, or "auto" if all columns should have the same width.

    .grid-container {
    display: grid;

    grid-template-columns: auto auto auto auto;
    or 
    grid-template-columns: 250px 250px // for 2 column .
    or 
    grid-template-columns : 200px 150px 300px 400px. // for 4 column 
    or 
    grid-template-columns: 80px 200px auto 40px;
    or
    grid-template-columns : 1fr 2fr 1fr // for 3 column 
    }

    Note: for same width for all column(box) use repeat fucntion grid-template-columns: repeat(3,1fr) // 3 columns of 1fr width

The justify-content property is used to align the grid items when they do not use all available space on the main-axis (horizontally).

        *start
        *end

Note: The grid item's total width has to be less than the container's width for the justify-content property to have any effect.
