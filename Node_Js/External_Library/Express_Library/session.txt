// 🔑 secret: "random stuff"

// Used to sign and encrypt the session cookie so users can't tamper with it
// In production, this should be a long, random, private string like "x9$kL#2mQ..." — never something obvious
// Think of it as a password for your sessions

// 💾 resave: false

// Controls whether the session is re-saved to the store on every request, even if nothing changed
// false = only save if the session was actually modified ✅ (recommended)
// true = save on every request, even unchanged ones (wasteful ❌)

// saveUninitialized: false

// Controls whether empty/new sessions (with no data) are saved
// false = don't save a session until something is actually stored in it ✅ (recommended)
// true = save a session for every visitor, even if they're not logged in (wastes storage ❌)



User visits site
      ↓
Server creates a session → stores it (memory/DB)
      ↓
Sends a cookie to browser with a session ID
      ↓
Browser sends cookie on every future request
      ↓
Server looks up the session ID → knows who the user is



// Set something in session (e.g., after login)
app.post("/login", (req, res) => {
    req.session.userId = 123; // saved on server
    res.send("Logged in!");
});

// Read from session
app.get("/dashboard", (req, res) => {
    if (req.session.userId) {
        res.send(`Welcome user ${req.session.userId}`);
    } else {
        res.send("Please login first");
    }
});


The cookie only stores the session ID — the actual data lives safely on the server.
